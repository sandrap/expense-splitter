## Component Visual Contracts

### Settings Panel (Tip & Tax)

**Position:** Below ItemsPanel, above ResultsPanel in the main content flow.

**Layout:** Single section with heading "Tip & Tax", containing two input groups stacked vertically with `space-y-4` (16px) gap.

#### Tip Input Group
- Label: "Tip" at body size (16px/400)
- Preset buttons: horizontal row of common percentages: 15%, 18%, 20%, 25%, Custom %
- Each preset button: `min-h-[44px] px-4 py-2 rounded-lg text-sm font-medium`
- Selected preset: `bg-blue-500 text-white`
- Unselected preset: `bg-gray-100 text-gray-600 dark:bg-gray-700 dark:text-gray-300`
- Container: `flex flex-wrap gap-2`
- "Custom %" opens an inline number input: `inputMode="decimal"`, width `w-20`, suffix "%" label
- Default selection: 18% (matches `settings.defaultTipPercent`)

**Note on TIP-01 (per-person tip):** The requirement states "User can set a tip percentage per person." The settings panel sets the default tip for everyone. Per-person override is handled in the Results panel -- each person's expanded breakdown includes an editable tip percentage that defaults to the global setting but can be overridden individually.

#### Tax Input Group
- Label: "Tax" at body size (16px/400)
- Single number input: `inputMode="decimal"`, width `w-20`, suffix "%" label
- Input style: `border border-gray-300 dark:border-gray-600 rounded-lg px-3 py-2 text-base bg-white dark:bg-gray-800`
- Default value: 0% (matches `settings.defaultTaxPercent`)
- Placeholder: "0"

### Results Panel

**Position:** Below Settings Panel. Visible as soon as there are people AND items in the store. Hidden otherwise.

**Layout:** Section with heading "Results" at heading size (20px/700).

#### Person Result Card
- Container: `border border-gray-200 dark:border-gray-700 rounded-lg` with `p-4`
- Background: `bg-gray-50 dark:bg-gray-800` (secondary surface)
- Stacked vertically with `space-y-4` (16px) gap between cards

**Collapsed state (default):**
- Layout: horizontal flex, three elements
  - Left: Person name at body size (16px/400)
  - Right: Total amount at display size (28px/700), formatted as "$XX.XX"
  - Expand chevron: right-pointing triangle (`>`) or down-pointing when expanded, `min-h-[44px] min-w-[44px]` touch target
- The total is the prominent element -- right-aligned, largest text on the card

**Expanded state (on tap/click of card or chevron):**
- Everything in collapsed state remains visible
- Below the summary line, a breakdown area appears with `mt-4 pt-4 border-t border-gray-200 dark:border-gray-700`
- Breakdown uses a definition-list style layout:

| Line | Left (Label 14px/400) | Right (Label 14px/400) |
|------|----------------------|----------------------|
| Per-item lines | Item description | Share amount "$X.XX" |
| Subtotal | "Subtotal" (font-medium) | "$XX.XX" (font-medium) |
| Tip | "Tip (XX%)" with inline editable percentage | "$X.XX" |
| Tax | "Tax" | "$X.XX" |
| Divider | Full-width `border-t` | |
| Total | "Total" (font-semibold) | "$XX.XX" (font-semibold, body 16px) |

- Each line: `flex justify-between py-1`
- Subtotal line has `mt-2 pt-2 border-t border-gray-100 dark:border-gray-700` to separate from item lines
- Per-person tip override: The "XX%" next to "Tip" is a clickable element that opens an inline input (same pattern as price inline edit from Phase 2). When overridden, show the custom percentage; when matching the global default, show the default.

#### Grand Total Row
- Below all person cards, separated by `mt-4 pt-4 border-t-2 border-gray-300 dark:border-gray-600`
- Layout: `flex justify-between`
- Left: "Grand Total" at heading size (20px/700)
- Right: Sum of all person totals at heading size (20px/700)
- This row serves as a sanity check for the user

## Interaction Contracts

### Tip Preset Selection Flow
1. User sees a row of preset tip buttons (15%, 18%, 20%, 25%, Custom %)
2. The 18% button is selected by default (accent background)
3. Tapping a preset instantly updates `settings.defaultTipPercent` and recalculates all results
4. Only one preset can be active at a time
5. Tapping "Custom %" deselects all presets and shows an inline number input
6. Typing in the custom input updates the tip percentage on blur or Enter

### Tax Input Flow
1. User sees a single text input for tax percentage, defaulting to 0
2. User types a number (e.g., "8.875")
3. On blur or Enter, value is parsed and `settings.defaultTaxPercent` is updated
4. Results recalculate immediately
5. If input is empty or invalid, revert to previous valid value

### Results Card Expand/Collapse Flow
1. All person result cards start collapsed (name + total only)
2. Tapping anywhere on the card toggles the expanded breakdown
3. Multiple cards can be expanded simultaneously
4. Expanded state shows item lines, subtotal, tip, tax, and total
5. Expand/collapse animates with a simple height transition (optional -- CSS `transition-all` or no animation)

### Per-Person Tip Override Flow
1. In the expanded breakdown, the tip line shows "Tip (18%)" by default
2. Tapping the percentage opens an inline number input (same pattern as Phase 2 inline edit)
3. User types a custom percentage for this specific person
4. On blur or Enter, the custom tip is applied to this person only
5. On Escape, the edit is cancelled and the global default is restored
6. The person's total recalculates immediately
7. Grand total row updates to reflect the change

### Live Recalculation
1. All results derive from `calculateResults(state)` which is a pure function of store state
2. Any change to people, items, tip, or tax triggers an automatic re-render
3. No "Calculate" button exists -- results are always current
4. This is a natural consequence of Zustand reactivity + derived `PersonResult[]`

## State Contracts

### New State Required
- Per-person tip overrides: `Map<personId, tipPercent>` or equivalent in the store (e.g., `tipOverrides: Record<string, number>`)
- Expanded card state: local component state (`useState<Set<string>>`) -- not in the global store since it is purely UI state

### Store Changes
- `BillSettings.defaultTipPercent` already exists (default 18) -- used as-is
- `BillSettings.defaultTaxPercent` already exists (default 0) -- used as-is
- New: `tipOverrides: Record<string, number>` in the store -- maps personId to custom tip percentage. When absent for a person, the global `defaultTipPercent` applies.
- New action: `setPersonTipOverride(personId: string, tipPercent: number)` -- sets a per-person tip
- New action: `clearPersonTipOverride(personId: string)` -- reverts to global default

### Calculation Engine Changes
- `calculateResults()` must accept per-person tip overrides and use them instead of `defaultTipPercent` when present for a given person

## Accessibility Notes

- Tip preset buttons use `role="radiogroup"` with individual `role="radio"` and `aria-checked` for screen readers
- Expand/collapse cards use `<button>` with `aria-expanded` attribute
- Expanded breakdown content uses `aria-labelledby` referencing the person name
- All inputs have visible labels or `aria-label` attributes
- Minimum touch target: 44x44px on all interactive elements (tip presets, expand toggles, inline edit triggers)
- Price/percentage inputs use `inputMode="decimal"` for mobile numeric keyboard
- Per-person tip override edit follows the same inline edit accessibility pattern from Phase 2 (focus management, Enter/Escape handling)
- Color is not the sole indicator of state -- selected tip preset has both color AND visual weight difference
