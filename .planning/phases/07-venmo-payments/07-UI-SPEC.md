### Component Visual Specs

**VenmoButton (inside PersonResultCard)**
- Element: `<a>` tag, not `<button>` -- it navigates to an external URL
- Position: Full-width below the name/amount row, above the expandable breakdown section
- Margin: `mt-2` (8px, token `sm`)
- Size: `w-full min-h-[44px] flex items-center justify-center gap-2`
- Background: `bg-blue-500 hover:bg-blue-600 dark:bg-blue-600 dark:hover:bg-blue-700`
- Text: `text-white text-sm font-bold`
- Shape: `rounded-lg`
- Interaction: `active:scale-95 transition-transform`
- Link behavior: `target="_blank" rel="noopener noreferrer"`
- Icon: Inline SVG Venmo "V" logo, 16x16, `fill="currentColor"` (white), placed before the label text
- Conditional render: Component returns `null` when `result.totalInCents === 0` -- no empty space, no disabled state

## Interaction Contracts

| Interaction | Behavior |
|-------------|----------|
| Tap "Pay with Venmo" | Opens `https://venmo.com/?txn=charge&note={encodedNote}&amount={amount}` in a new tab. On mobile with Venmo installed, the OS intercepts and opens the Venmo app. On desktop or without Venmo, user lands on venmo.com. |
| Person owes $0.00 | Button is not rendered. No empty space remains. Card looks identical to current design when total is zero. |
| Hover on button (desktop) | Background transitions from `blue-500` to `blue-600` (light) or `blue-600` to `blue-700` (dark). |
| Active press on button | `scale-95` transform for tactile feedback (matches project convention). |

## URL Construction Contract

The `buildVenmoUrl` function constructs the URL as follows:

| Parameter | Value | Source |
|-----------|-------|--------|
| Base URL | `https://venmo.com/` | Web-compatible format that triggers native app on mobile |
| `txn` | `charge` | Always "charge" -- the bill splitter user is requesting money |
| `amount` | `(totalInCents / 100).toFixed(2)` | e.g., `42.50` for 4250 cents |
| `note` | `encodeURIComponent(billName \|\| 'Split bill')` | Bill name from Zustand state; fallback "Split bill" when empty |
| `recipients` | omitted | CONTEXT.md decision: no recipient username collection |

Example output: `https://venmo.com/?txn=charge&note=Dinner%20at%20Olive%20Garden&amount=42.50`

Source: CONTEXT.md locked decisions on note content, fallback text, amount formatting. Web URL format confirmed via research -- `https://venmo.com/?txn=charge&note=...&amount=...` works without a username; user selects recipient in the Venmo app.

## State Descriptions

| State | What the User Sees |
|-------|--------------------|
| Person owes money | PersonResultCard shows the blue "Pay with Venmo" button below the total amount, full width. |
| Person owes $0.00 | PersonResultCard shows no Venmo button. Card layout is unchanged from current behavior. |
| Bill has a name | Venmo opens with the bill name as the note (e.g., "Dinner at Olive Garden"). |
| Bill has no name | Venmo opens with "Split bill" as the note. |
| Venmo app installed (mobile) | OS intercepts the `https://venmo.com/` link and opens the Venmo app directly with amount and note pre-filled. New tab opens but may be blank or redirect to venmo.com. |
| Venmo app not installed | Browser opens venmo.com in a new tab. User can complete the action on the Venmo website or install the app. |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Tap target | `min-h-[44px]` on the anchor element (project standard) |
| Screen reader | `aria-label="Pay {person name} {formatted amount} with Venmo"` -- descriptive label including person name and amount for context when navigating by links |
| External link | `target="_blank" rel="noopener noreferrer"` -- standard external link pattern |
| Color contrast | White text on `blue-500` background: 3.8:1 ratio. Add `font-bold` to meet WCAG AA for large text (14px bold qualifies). Blue-600 dark mode background improves to 4.3:1. |
| Focus visible | `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-blue-500` for keyboard navigation |
