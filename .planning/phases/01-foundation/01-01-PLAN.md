  <action>
    1. Create `src/types/models.ts` with these exact interfaces:

       ```typescript
       export interface Person {
         id: string;
         name: string;
       }

       export interface Item {
         id: string;
         description: string;
         priceInCents: number;
         splitMode: 'shared' | 'assigned';
         assignedTo: string[];
       }

       export interface BillSettings {
         defaultTipPercent: number;
         defaultTaxPercent: number;
       }

       export interface AppState {
         people: Person[];
         items: Item[];
         settings: BillSettings;
       }

       export interface ItemLine {
         itemId: string;
         description: string;
         shareInCents: number;
       }

       export interface PersonResult {
         personId: string;
         name: string;
         itemLines: ItemLine[];
         subtotalInCents: number;
         tipInCents: number;
         taxInCents: number;
         totalInCents: number;
       }
       ```

       KEY DECISIONS (locked, non-negotiable):
       - `Item.splitMode` is an explicit enum ('shared' | 'assigned') -- NOT inferred from assignedTo length
       - `PersonResult` is derived, never stored in state
       - `priceInCents` is always integer cents
       - `Person` is minimal (id + name only) -- no tipPercent field yet

    2. Create `src/store/billStore.ts` implementing the Zustand v5 store:

       ```typescript
       import { create } from 'zustand';
       import type { Person, Item, BillSettings } from '../types/models';
       ```

       State shape:
       - `people: Person[]` (initially empty)
       - `items: Item[]` (initially empty)
       - `settings: BillSettings` (initially `{ defaultTipPercent: 18, defaultTaxPercent: 0 }`)

       Named actions (all must be exported on the store):
       - `addPerson(name: string)` -- creates Person with `crypto.randomUUID()` id
       - `removePerson(id: string)` -- removes from `people[]` AND filters the person's id out of every item's `assignedTo[]` array (CRITICAL: prevents stale references per Pitfall 3)
       - `updatePerson(id: string, updates: Partial<Person>)` -- merges updates into matching person
       - `addItem(description: string, priceInCents: number)` -- creates Item with UUID, default `splitMode: 'shared'`, empty `assignedTo: []`
       - `removeItem(id: string)` -- removes from `items[]`
       - `updateItem(id: string, updates: Partial<Item>)` -- merges updates into matching item
       - `updateSettings(updates: Partial<BillSettings>)` -- merges into settings

       IMPORTANT: Use `import { create } from 'zustand'` (named import, NOT default -- Zustand v5 removed default exports).
       IMPORTANT: Use `create<BillState>()(...)` with the double-parenthesis TypeScript pattern.
  </action>
