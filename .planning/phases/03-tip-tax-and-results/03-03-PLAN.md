1. In `src/components/SettingsPanel.tsx`:

   **Fix isCustom initialization** — change from:
   ```typescript
   const [isCustom, setIsCustom] = useState(false)
   ```
   to:
   ```typescript
   const [isCustom, setIsCustom] = useState(() => !([15, 18, 20, 25] as readonly number[]).includes(settings.defaultTipPercent))
   ```
   This ensures that if `defaultTipPercent` is already a non-preset value (e.g., 22), the Custom input shows on mount. Use `PRESETS.includes(settings.defaultTipPercent as any)` if PRESETS is `as const` (since readonly arrays need type assertion for `.includes`).

   **Harden custom tip input commit** — the blur/Enter handler must reject negative values AND cap at a reasonable maximum. Change the commit logic to:
   ```typescript
   const val = parseFloat(customDraft);
   if (!isNaN(val) && val >= 0 && val <= 100) {
     updateSettings({ defaultTipPercent: val });
   } else {
     // Revert draft to current setting
     setCustomDraft(String(settings.defaultTipPercent));
   }
   ```
   The `val <= 100` cap prevents absurd tip percentages. Empty string parses to NaN, so it is handled by the isNaN check.

   **Harden tax input commit** — same pattern for the tax blur/Enter handler:
   ```typescript
   const val = parseFloat(taxDraft);
   if (!isNaN(val) && val >= 0 && val <= 100) {
     updateSettings({ defaultTaxPercent: val });
   } else {
     setTaxDraft(String(settings.defaultTaxPercent || ''));
   }
   ```
