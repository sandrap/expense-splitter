## Interaction Contract

### Share Flow

1. User taps Share icon (header) or "Share this split" button (Results)
2. App encodes current bill state (billName + people + items + settings + tipOverrides) into compressed URL
3. **If Clipboard API available:** Copy URL to clipboard, show toast "Link copied!" for 2 seconds
4. **If Clipboard API unavailable:** Show modal with URL in selectable text area

### URL Load Flow

1. On app mount, check `window.location` for encoded bill state in URL params
2. **If valid encoded state found:** Decode, hydrate Zustand store one-shot, render bill normally
3. **If malformed or undecodable:** Silent fallback to empty bill state (no error shown)
4. **If no URL params:** Normal empty bill state (no change from current behavior)
5. URL stays in browser address bar unchanged after load
6. Loaded bill is fully editable (identical behavior to manually-created bill)

### Bill Name Edit Flow

1. User taps placeholder "Tap to name this bill" or existing bill name text
2. Input appears with current value (or empty for unnamed), autoFocus, text selected
3. User types name
4. Blur or Enter: trim whitespace, commit to store (empty after trim = unnamed)
5. Escape: cancel edit, revert to previous value

## Edge Case Visual Contract

| Scenario | Expected Visual State |
|----------|----------------------|
| Unnamed bill + Share tapped | URL encodes state without billName (or with empty string); toast shows normally |
| Named bill + Share tapped | URL encodes state including billName; toast shows normally |
| Very long bill name | Truncated with ellipsis in display mode (truncate class, max-w-xs); full text visible in edit mode (input scrolls horizontally) |
| Clipboard API unavailable | Modal appears instead of toast; URL displayed in selectable text area |
| Empty bill (no people, no items) + Share tapped | Share still works (encodes empty state); toast shows normally. The "Share this split" button in Results is not visible (gated on results existing) |
| Shared URL opened on fresh browser | Bill loads with all data intact; bill name shows in header if present |
| Malformed/truncated shared URL | Silent fallback to empty bill; no error message; app behaves as normal fresh load |
| Share icon tapped rapidly | Debounce not required (encode + clipboard is synchronous-fast); each tap overwrites clipboard |
