    6. Implement `calculateResults(state: AppState): PersonResult[]`:

       Algorithm:
       a. If state.people is empty, return [].
       b. Initialize a map: personId -> { itemLines: [], subtotalInCents: 0 }.
       c. For each item:
          - Resolve the person list:
            - If splitMode === 'shared' and assignedTo is empty: all people
            - If splitMode === 'shared' and assignedTo is non-empty: those specific people
            - If splitMode === 'assigned' and assignedTo is non-empty: those specific people
            - If splitMode === 'assigned' and assignedTo is empty: skip (no one gets charged)
          - Distribute item.priceInCents among resolved people using largestRemainderDistribute(priceInCents, count)
          - Add an ItemLine to each person's accumulator with their share
          - Add each share to the person's subtotalInCents
       d. For each person, compute:
          - tipInCents = Math.round(subtotalInCents * settings.defaultTipPercent / 100)
          - taxInCents (proportional): use distributeProportional to split total tax across people by their subtotal weights
            - totalTaxableCents = sum of all subtotals
            - totalTaxCents = Math.round(totalTaxableCents * settings.defaultTaxPercent / 100)
            - Distribute totalTaxCents proportionally by each person's subtotalInCents
          - totalInCents = subtotalInCents + tipInCents + taxInCents
       e. Return PersonResult[] with all fields populated.

       CRITICAL FORMULA (locked decision):
       - tip = subtotal * tipRate (pre-tax only)
       - tax = proportional share of total tax
       - total = subtotal + tax + tip

       CRITICAL GUARDS:
       - Division by zero: empty assignedTo with 'assigned' mode -> skip item
       - Zero people -> return []

    7. Run tests -- must PASS.

    8. Review for clarity. Ensure no floating-point arithmetic exists in the engine (all integer cents). Run tests again.
