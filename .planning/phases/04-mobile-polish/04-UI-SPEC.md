# Phase 4 — UI Design Contract

> Visual and interaction contract for the mobile polish phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, tight padding (px-1, py-1) |
| sm | 8px | Compact element spacing, chip gaps (gap-2, py-2, mt-2) |
| md | 12px | Card internal padding (p-3, px-3) |
| lg | 16px | Section padding, panel padding (p-4, space-y-4) |
| xl | 32px | Major section breaks (space-y-8) |

Exceptions: 44px minimum touch target dimension (min-h-[44px], min-w-[44px]) for all interactive elements. This is a mobile accessibility requirement, not a spacing token.

## Dark Mode Contract

| Property | Value |
|----------|-------|
| Strategy | Tailwind `media` strategy (`prefers-color-scheme`) |
| Toggle | None — auto-follows system preference |
| Implementation | Add `@media (prefers-color-scheme: dark) { .dark }` logic OR set Tailwind darkMode to `media` |
| Existing coverage | All components already have `dark:` class variants |
| Gap | The `dark` class is not currently wired to the `<html>` element or Tailwind `media` strategy is not configured |

## Touch Target Contract

Minimum interactive element size: 44x44px (WCAG 2.5.8 Target Size).

| Element | Current State | Required Change |
|---------|--------------|-----------------|
| Remove Person button | min-h-[44px] min-w-[44px] | None (already compliant) |
| Tip preset buttons | min-h-[44px] | None (already compliant) |
| AssignmentChips | min-h-[44px] min-w-[44px] | None (already compliant) |
| Expand/collapse arrow (PersonResultCard) | min-h-[44px] min-w-[44px] | None (already compliant) |
| Click-to-edit person name (PersonRow) | No min size | Add min-h-[44px] and sufficient padding |
| Click-to-edit item description (ItemRow) | No min size | Add min-h-[44px] and sufficient padding |
| Click-to-edit item price (ItemRow) | No min size | Add min-h-[44px] and sufficient padding |
| Click-to-edit tip % (PersonResultCard) | No min size | Add min-h-[44px] with inline padding |
| Shared/Assigned toggle buttons (ItemRow) | No min size | Add min-h-[44px] |
| Remove Item button | min-h-[44px] (no min-w) | Add min-w-[44px] for horizontal target |

## Keyboard Occlusion Contract

| Property | Value |
|----------|-------|
| Trigger | `focus` event on any input with `inputMode="decimal"` or `inputMode="text"` |
| Behavior | Active input scrolls into visible viewport above the software keyboard |
| Method | `element.scrollIntoView({ block: 'center', behavior: 'smooth' })` on focus, combined with CSS `scroll-padding-bottom: 40vh` on the scroll container |
| Priority fields | Item price inputs, per-person tip override inputs, tax % input, custom tip % input |
| Fallback | If `visualViewport` API is available, listen to `resize` events to detect keyboard appearance and adjust scroll position |

---

## Live Recalculation Contract

| Property | Value |
|----------|-------|
| Trigger | Every keystroke (`onChange`) in any numeric input |
| Debounce | None — immediate on each keystroke |
| Invalid input handling | Treat empty string, ".", "1.", and NaN as 0 cents for calculation purposes |
| Draft display | Local component state holds the raw string; display shows calculated result using 0-fallback |
| Commit | On `blur` or `Enter`, validate and persist to Zustand store |
| Cross-card update | A per-person tip draft change triggers full `calculateResults()` recalculation affecting all cards |
| Architecture | Draft values flow up via callback props to a parent component (or custom hook) above ResultsPanel that merges drafts before calling `calculateResults()` |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add Person" / "Add Item" (existing, unchanged) |
| Empty state — no data | "No results yet" / "Add people and items above to see what everyone owes." |

| Empty state — items only | "Add people to split the bill with." |
| Unassigned warning | "Not assigned to anyone" |
| Price validation error | "Enter a valid price (e.g. 12.50)" |
| Hint — no people for chips | "Add people first to assign items" |

| Destructive: Remove Item | "Remove" (inline button, no confirmation dialog — action is easily reversible by re-adding) |

No new copywriting is introduced in Phase 4. All copy above is already implemented and confirmed correct.

## Edge Case Visual Contract

| Scenario | Expected Visual State |
|----------|----------------------|
| Zero people + zero items | ResultsPanel shows "No results yet" heading + "Add people and items above to see what everyone owes." body |
| People added, no items | ResultsPanel shows "Results" heading + "Add items to the bill to calculate results." body |
| Items added, no people | ResultsPanel shows "Results" heading + "Add people to split the bill with." body |
| All items at $0.00 | ResultsPanel shows $0.00 per person, no error state |
| Single person | ResultsPanel shows one PersonResultCard naturally |
| All items in assigned mode with no assignees | Items show amber border + "Not assigned to anyone" warning; ResultsPanel shows $0.00 per person (skip-item engine guard) |
| Invalid draft input mid-typing | Results show $0.00 for that value (0-fallback), no error message until blur/commit |
